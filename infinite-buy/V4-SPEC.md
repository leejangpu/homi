## 0. 전체 구조

V4.0 = **일반모드** + **소진후 리버스모드** 2-상태 머신.

- **일반모드**: 소진 전(쿼터매도 포함)의 일반 매매 과정.
- **리버스모드**: T값이 1회분 미만 남은 시점부터, 특정 조건으로 일반모드 복귀 전까지의 청산 과정.
- 종목: TQQQ, SOXL (그 외 비권장).
- 분할: 20 / 30 / 40 분할. (예시는 대부분 40분할)
- 추가입금 금지: 진행 중 외부 자금 추가 투입 안 함.

## 1. 별%(별지점) — [확정]

별지점 = 매수/매도를 가르는 기준값 (V2.2부터의 개념).

```
별%(percent) = target − T × (target × 2 / 분할수)
  · target: TQQQ = 15, SOXL = 20
  · 일반화: 20분할 TQQQ (15−1.5T)% · 40분할 TQQQ (15−0.75T)%
           20분할 SOXL (20−2T)%   · 40분할 SOXL (20−T)%

별지점가 = 평단 × (1 + 별%/100)            (소수 2자리 반올림)
매수점   = 별지점 − 0.01                    (매수·매도 겹침 방지)
매도점   = 별지점 (그대로)
```

T가 분할수 절반을 지나면 별%가 음수 → 별지점이 평단 아래(후반전).

**기대값 앵커** (문서 p3-4 예시): 20분할 SOXL, 평단 38.30, T=8.6
→ 별% = 20 − 2×8.6 = 2.8% → 별지점 = 38.30 × 1.028 = **39.37**

## 2. 1회매수금 — [확정]

```
1회매수금 = 잔금 / (분할수 − T)
  · 20분할: 잔금 / (20 − T)
  · 40분할: 잔금 / (40 − T)
  · 잔금 = 원금 − 누적 매수금 (당일 시점의 남은 현금)
```

- 정상 진행(전량체결) 시 1금 달러값은 일정하게 유지되고, 덜 체결되면 자동 증가하여 시드를 끝까지 소진.
- 분모(분할수−T) → 0 ⇒ 리버스모드 진입.

**기대값 앵커** (문서 p5 예시): 원금 20000, 40분할
→ 1일차 1금 = 20000/40 = 500. 478 체결 가정 → 잔금 19522, T=1
→ 2일차 1금 = 19522 / (40−1) = 19522/39 = **500.5641…**

## 3. T값 갱신 — [확정] (이전 버전과 다름: 누적금액식 아님, 이벤트 누적식)

### 일반모드
```
1회매수 전량 체결 → T = 직전T + 1
1회매수 절반 체결 → T = 직전T + 0.5
쿼터매도(익절) 발생 → T = 직전T × 0.75
```
- T는 소수점 무제한 (예: 5.165479525).
- ⚠️ **모든 분류는 KIS 실제 체결내역(`getOrderHistory`)으로 판정. 추측 금지.**

**기대값 앵커** (문서 p2): T=7 → 전량 T=8 · 절반 T=7.5 · 쿼터매도 T=7×0.75=5.25

### 리버스모드 (문서2 p5)
```
20분할: 매도시 T = 직전T × 0.9   · 매수시 T = 직전T + (20 − 직전T) × 0.25
40분할: 매도시 T = 직전T × 0.95  · 매수시 T = 직전T + (40 − 직전T) × 0.25
```
**기대값 앵커**: 40분할 소진후 T=39.5 → 첫날 매도 T=37.525
→ 둘째날 쿼터매수 T = 37.525 + (40−37.525)×0.25 = **38.14375**
- 이 T는 리버스 종료 후 일반모드 복귀 시 그대로 승계.

> **[확인필요]** 같은 날 매수체결과 쿼터매도가 동시 발생 시 T 갱신 순서/우선순위는 문서에 없음.

## 4. 일반모드 매수 — [확정 골격 + 큰수매]

매수는 **전부 LOC**. 매일 재계산하여 걺.

### 4.1 처음매수 (T=0, 보유 0)
- 무조건 매수 의도. **큰수매**로 전날종가 **+10%** 자리에 LOC 시작 (큰수매 §6).
- 그 아래로 1주씩 LOC 사다리.

### 4.2 전반전 (T=1 ~ 분할수/2 미만, 40분할이면 T=0~19.99…)
- 1회매수금의 **절반 → 별지점 LOC**, **절반 → 평단 LOC**.
- 큰 하락 대비, 아래로 1주씩 LOC 사다리 추가(1회매수금 소진 목적).

### 4.3 후반전 (분할수/2 이상 ~ 분할수−1, 40분할이면 T=20~39)
- 1회매수금 **전액 → 별지점 LOC**. 후반전은 별지점 < 평단.
- 아래로 1주씩 LOC 사다리 추가.
- T가 39 초과(분할수−1 초과)면 리버스모드로 전환.

> **[예시기반/확인필요]** 별지점·평단의 정확한 주문 "수량 분배"와 하방 사다리 단들의 "가격 간격"은
> 문서가 예시 표(언이시트 Excel 산출)로만 보여주고 닫힌 공식이 없음. §6의 큰수매 합산 규칙과
> "현재가+15% 고정"으로 자동화 시 수량은 결정되나, 하방 사다리 단가 간격 규칙은 별도 확인 필요.

## 5. 일반모드 매도 — [확정] (현재 v3.0 코드와 동일 구조)

매도는 매일 걺. 전·후반 공통:
```
보유의 1/4 (올림, 최소 1) → 별지점(=별%) LOC 매도   ← 쿼터매도
나머지 3/4              → TQQQ +15% / SOXL +20% 지정가(LIMIT) 매도
```
- 매수는 LOC, **최종(3/4) 매도는 지정가(LIMIT)**. (LOC 아님)
- 지정가는 프리장 시작(서머 17시/비서머 18시)에 걸기 권장. 주간장 미사용.
- 쿼터매도(별지점 LOC 매도)가 체결되면 T = 직전T × 0.75 (§3).

## 6. 큰수 매수 — [확정] (출처: 큰수매 PDF)

**목적**: 증권사는 전일종가 대비 약 ±20% 벗어난 주문을 오류로 거부 → 큰 하락 시 별지점/평단
주문이 거부됨. "별가격 아래에서 무조건 매수"를 보장하기 위한 장치.

### 두 가지 타이밍
1. **새 사이클 시작**: 전날종가 **+10%** 에 LOC 매수 시작.
2. **사이클 중 큰 하락** (별%/평단이 전일종가보다 ~20%+ 위): 전날종가 **+5~20%**(가이드 +15%)
   자리에 큰수 LOC.

### 합산 규칙 [확정]
```
큰수 자리 가격 = 전일종가 × (1 + p),  자동화 표준 p = 15% 고정  (댓글 "언이시트 알림2")
큰수 자리 수량 = (큰수 자리 가격 이상에 걸렸을) 모든 매수 단들의 수량 합
그 아래로     = 각 단에 1주씩 LOC
```
예 (PDF p3-4): 평단 39.71, 3일 연속 하락으로 전일종가 30 →
33.33$(≈30×1.111)에 큰수매: 41.34의 12개 + 39.71의 13개 + … 33.33까지 합 = **30개**를 33.33에 LOC,
그 아래 32.25에 1개·31.25에 1개… 1주씩.

### 가변 원칙 [확정]
- p(%)는 고정값이 아닌 가이드. **원칙은 "별가격 아래 무조건 매수"**.
  - 갭상승 시 더 높게(+20%↑), 갭하락 시 더 낮게(+5~10%). 자동화는 +15% 고정 채택.

### 매도거부 [확정]
- 큰 하락장에서 매도는 증권사가 어차피 거부 → **걸지 않아도 됨. 매도점을 바꿔 걸면 안 됨.**

## 7. 소진후 리버스모드 — [확정] (출처: 문서2)

### 7.1 발동조건
- 20분할 T>19 / 40분할 T>39 (1회분 미만 잔금) 시점부터. 남은 잔금도 함께 사용.

### 7.2 매도 과정
- **첫날**: 전체수량을 10등분(20분할)/20등분(40분할)하여 **MOC 매도**(무조건). 나눠떨어지지 않으면 내림.
- **둘째날 이후**: 별지점 위에서, **직전 보유수량 ÷ (10 또는 20)** 수량을 LOC 매도.
  - 예: 200개 → 첫날 10등분 20개(MOC) → 190 → 둘째날 9개 → 181 → 9개 → 172 → 8개 …

### 7.3 매수 과정 (쿼터매수)
- 둘째날부터 `(남은잔금 + 당일매도금) / 4` 금액을 **별지점 아래 LOC** 매수.
  - 예: 소진시 잔금 400 + 첫날 MOC 매도금 300 = 700 → 700/4 = 175를 둘째날 매수 시도.

### 7.4 별지점 정의 (리버스) [확정]
- 별지점 = **직전 5거래일 종가의 평균**. (월요일 본장 직전이면 전주 월~금 종가 평균)
- 첫날은 무조건 MOC라 별지점 계산 없음. 둘째날부터 별지점 기준 매수/매도 분리.

### 7.5 T값 (리버스) — §3 참조.

### 7.6 종료조건 [확정]
- 종가가 보유평단 대비 **TQQQ −15% / SOXL −20% 위로 회복**하는 것을 확인한 다음날부터 일반모드 복귀.
  (−15/−20%는 일반모드 마지막 T의 쿼터매도 기준과 동일. 단 하루 만에 종료될 수도.)
  - 예: 평단 40, SOXL → 종가가 32(=40×0.8) 위로 올라오면 종료.
- 일반모드 복귀 시 T 그대로 승계, 1금 = 잔금/(분할수−T).
- 복귀 후 진행 중 또 T가 1회분 미만 남으면 다시 리버스모드 시작.
- 보유수량 0 → 사이클 종료(→ 다음 사이클, 복리 여부 선택).

## 8. 백테스트 체결 모델 (오프라인 결정론적 시뮬)

KIS 모의투자(VTS) 서버는 현재 코드에 없고, "기대값과 온전히 일치"는 결정론적 시뮬에서만 가능.
→ yfinance 과거 OHLC(SOXL/TQQQ 실데이터)를 일별 투입해 체결 재현.

```
LOC 매수 : 종가 ≤ 지정가  → 종가로 체결
LOC 매도 : 종가 ≥ 지정가  → 종가로 체결
MOC      : 종가로 무조건 체결
지정가(LIMIT) 매도 : 당일 고가 ≥ 지정가 → 지정가로 체결
체결 분류(T 갱신) : 당일 1회매수금 대비 체결액(전량/절반) · 쿼터매도(별지점 매도) 체결 여부
```

> **[확인필요]** 증권사 ±20% 거부를 백테스트에서 모델링할지 여부 — §6 큰수매의 합산 결과가
> 일반 사다리와 체결 수량이 달라지는 유일한 지점. 문서 충실성을 위해 거부 밴드를 모델링하고
> 큰수매 합산을 적용하는 것을 기본으로 한다(확인 후 확정).

## 9. 미확정 항목 → ✅ 언이시트 Apps Script로 전부 확정

언이시트 스크립트(updateDaily) 입수로 아래 모두 확정. `src/v4/unisheet.ts`가 스크립트를 1:1 포팅함.

1. **하방 LOC 사다리(대폭락 티어)** — `tierPrice = amt / m` (m=baseQty+1…), `tierPrice < 큰수`인 동안, **최대 5단**, 1주씩. (floor 아님, toFixed(2) 반올림)
2. **별지점/평단 수량 분배(전반전)** — `q1 = floor((amt×0.5)/별지점매수가)`, `totalQty = floor(amt/평단)`, `q2 = max(0, totalQty − q1)`. 가격은 각각 `min(별지점매수가, 큰수)`, `min(평단, 큰수)`.
3. **동일일 매수+매도 T 순서** — 스크립트는 보유수량 변화로 action(BUY/SELL/NONE) 단일 판정. 매도시 3상황(100%/≤60%/일반쿼터)으로 분기. `src/v4/unisheet.ts` 참조.
4. **±20% 거부 밴드** — 별도 모델링 불필요. 큰수(=종가×1.10) 합산 주문으로 거부 상황을 흡수(가격을 `min(별/평단, 큰수)`로 캡).

**확정된 추가 규칙(스크립트)**:
- 쿼터 수량 = `floor(보유×0.25)` (올림 아님 — 4주 미만이면 0).
- 1회매수금 amt = `balance/max(0.1, n−T)` (NORMAL) / `balance/4` (REVERSE) / `principal/n` (새사이클).
- 리버스 T: 매도 `oldT×(1−2/n)`(모든 n), 매수 `oldT+(n−oldT)×0.25`. 리버스 진입 1일차 매수금지+MOC 매도.
- 리버스 종료: `종가 > 평단×(1−목표%/100)`.
- 잔금 = `원금 − 보유×평단 + 누적실현손익 + 당일손익`.
