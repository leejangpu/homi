# PRD (Product Requirements)

## 1. 제품 비전
가족 구성원이 각자 개인 도구를 사용하면서도, 필요한 데이터는 공유 가능한 "가정용 통합 웹앱"을 만든다.

## 2. 사용자
- 현재: 부부 2명
- 미래: 가족 구성원 확장 가능

## 3. 핵심 요구사항
1. 데이터 자체 관리
- 데이터는 git 기반 파일로 관리
- 외부 SaaS 종속 최소화

2. 계정/인증
- 구성원별 개별 계정 필수
- 로그인 후 권한에 맞는 데이터만 접근

3. 모듈형 구조
- 가계부, ToDo, 일기 같은 기능을 "모듈"로 제공
- 사용자 페이지에 모듈을 자유롭게 추가/배치

4. 개인/공유 공간
- 개인 공간: 본인 전용 모듈
- 공유 공간: 다수 구성원이 접근 가능한 모듈

5. 다중 인스턴스
- 동일 기능(예: 가계부)을 여러 개 생성 가능
- 각 인스턴스는 독립된 데이터/설정 보유

6. AI 확장성(후속)
- 자연어로 데이터 조회/수정

7. 분석 확장성
- 원본 데이터(RAW) 보존
- 통계/차트/리포트 등 2차 변환 가능 구조

## 4. 비기능 요구사항
- 확장성: 사용자/모듈 증가에 유연해야 함
- 안정성: 권한 누락으로 인한 데이터 노출 방지
- 추적성: 변경 이력(생성/수정 시각, 작성자) 유지
- 유지보수성: 모듈 추가 시 기존 코드 영향 최소화

## 5. MVP 범위
- 인증(회원가입/로그인/로그아웃)
- 개인/공유 공간 생성
- 모듈 인스턴스 생성/목록/삭제
- 기본 모듈 3종(가계부, ToDo, 일기)
- 간단한 통계(합계, 건수, 월별 추이)
