# 떨사오팔 (ddsaopal) 사양서

> **이름 뜻**: **떨**어지면 **사**고 **오**르면 **팔**다.
> 이 문서가 전략의 **권위 사양(단일 진실원)**이다. 코드(`src/`)는 이 문서를 포팅한 것이며, 충돌 시 이 문서가 우선한다.
> 상태: ✅ 최초 확정 (2026-07-23). 사용자 결정 3건 반영 완료.

## 1. 한 줄 요약

예수금을 n분할(기본 7) 하고, **전일 종가보다 떨어진 날에만** 1분할씩 LOC로 매수한다. 매수한 각 로트("떨")는 **매수가 +0.3%**에 LOC로 매도한다. 매수 후 **12영업일**이 지난 로트는 종가매도(손절)한다. 보유 로트가 0이 되면(flat) 한 사이클이 끝나고, 남은 예수금으로 다시 n분할해 재시작한다.

## 2. 용어

| 용어 | 뜻 |
|---|---|
| **종목** | 단일 US 티커 (`config.symbol`). 다종목은 향후 확장. |
| **n / 분할수** | `config.splits`, 기본 7. |
| **떨(lot)** | 1회 매수로 생긴 보유 단위. `{ id, buyDate, buyPrice(체결가), qty, daysHeld }`. |
| **1분할금액 (splitAmount)** | **사이클 시작 시점에** `사이클시작예수금 / n` 으로 **고정**되는 1회 매수 상한. |
| **사이클** | 보유 떨 0(flat)에서 시작 → 다시 flat 이 되면 종료. |
| **prevClose** | "전일 종가". 매수 기준선. 매일 갱신된다(내 보유가와 무관). |

## 3. 통화·시장

- 미국 주식, **USD 예수금** 기준. 분할·매수금액 모두 달러.
- 모든 주문은 **종가 체결(CLS)** 계열이라 하루 1회, 미국 정규장 마감 시점에 체결된다.
- 개장/휴장 판정은 토스 market-calendar(US)로 한다.

## 4. 주문 타이밍 모델 (중요)

모든 주문이 `timeInForce=CLS`(장 마감 체결)이므로, 하루 흐름은:

- **main-close** (미국 장 **마감 후**, KST 아침): 오늘 제출했던 주문의 체결을 확인해 상태를 갱신하고, **다음 개장일의 주문 계획**을 계산·영속한다. prevClose = 오늘 종가.
- **main-open** (미국 장 **개시 전**, KST 새벽): main-close가 저장해 둔 **계획을 그대로 제출**만 한다(추가 판단 없음, 게이트만 검사).

즉 "판단은 마감 후, 제출은 개장 전"이다. (기존 무한매수법 v4와 동일 구조.)

## 5. 매수 규칙

1. **기준선 = prevClose(전일 종가).** 매 개장일 LOC 매수를 `price = prevClose`, `side=BUY`, `timeInForce=CLS`로 건다.
   - LOC 매수는 **당일 종가 ≤ prevClose 이면 당일 종가에 체결**된다. → "전일보다 떨어지면 산다"가 자동 성립. 안 떨어지면 미체결·소멸(다음날 새로 건다, 상태 이월 없음).
2. **매수금액** = `min( splitAmount, 매수가능금액 / 남은분할수 )`.
   - `남은분할수 = n − 현재 보유 떨 개수`. (매도·손절로 떨이 빠지면 다시 늘어난다.)
   - 평소엔 `splitAmount`(상한)로 매수하고, 손절이 이어져 현금이 쪼그라들면 자연히 작아진다("min으로 매수").
   - `매수가능금액`은 내부 장부가 아니라 **토스 매수가능금액 API**로 조회(수수료·환·결제지연 반영).
3. **수량** = `floor(매수금액 / prevClose)`. 0이면 그날 매수 없음(자연 패스).
4. **남은분할수 ≤ 0** (이미 n개 보유) 이면 매수 주문을 걸지 않는다. 매도로 한 칸이 비면 그때 다시 매수 가능.
5. 체결되면 로트를 기록: `buyPrice = 체결가(= 당일 종가)`, `qty = 체결수량`, `daysHeld = 0`.
   - 부분체결이면 체결된 수량만 로트로 기록.

## 6. 매도 규칙 (이익 실현)

- 보유한 **각 로트마다** `price = ceilTick(buyPrice × 1.003)`, `side=SELL`, `timeInForce=CLS` LOC 매도를 건다.
  - `ceilTick`: 틱에 맞춰 **올림**(절삭이 아니라 올림)해야 실제 수익률이 0.3% 이상 보장된다.
  - US 틱: $1 이상 → 소수 2자리, $1 미만 → 소수 4자리.
- 하루 제출 주문 = (매수 계획 1건, 있으면) + (보유 로트 수만큼 매도).
- 급등해 종가가 여러 로트의 매도가를 모두 넘으면 **그 로트들이 모두 당일 종가에 체결**된다(각각 독립).
- 로트가 매도되면 `남은분할수`가 늘어 다음날 다시 매수 여지가 생긴다.
- **예외**: 그 로트가 다음날 손절 대상이면(§7) 이익매도 대신 **종가매도**만 건다(같은 주식 이중매도 방지).

## 7. 손절 규칙 (12영업일 종가매도)

- 각 로트는 **매수 다음날부터 1일차**로 세어 **12일차(개장일)에 종가매도**한다.
  - 카운팅: 매수 체결일 = 0. 이후 마감이 지날 때마다 `daysHeld += 1`. `daysHeld == 12`가 되는 날 종가에 매도되도록, **다음날 계획 시 `daysHeld + 1 >= 12`인 로트**를 종가매도 대상으로 잡는다.
- **토스는 MOC(시장가 종가주문)를 지원하지 않는다** (`CLS`는 `LIMIT`과만 결합 가능). 따라서 종가매도는 **아주 낮은 지정가 LOC 매도**로 구현한다.
  - 매도 LOC는 종가 ≥ 지정가면 종가에 체결되므로, 지정가를 충분히 낮게 두면 사실상 MOC와 동일하게 종가 체결된다.
  - 지정가 = `floorTick( prevClose × (1 − stopSellDiscount) )`, `stopSellDiscount` 기본 0.30.
  - ⚠️ 트레이드오프: 당일 종가가 지정가보다 더 떨어지면(예: 30% 이상 급락 마감) 미체결된다. 그 경우 다음날 `daysHeld+1`이 더 커진 채 다시 손절 대상으로 재시도된다. 토스 가격 밴드(주문 거부) 여부는 실거래 검증 필요 → §10.
- 손절 체결분은 현금으로 회수되어 다음 매수 재원이 된다.

## 8. 사이클 규칙

- **사이클 시작**: 보유 떨 0(flat) 상태에서 새 매수가 나갈 때. 시작 순간 `splitAmount = 매수가능금액 / n`으로 **고정**한다.
- **사이클 종료(리셋)**: 마감 반영 후 보유 떨이 0이 되면 사이클 종료. 다음 사이클은 남은 예수금으로 다시 n분할해 재시작.
- "n분할 전부 소진(=n개 보유)"은 **최대 진입 상태**일 뿐 종료가 아니다. 매도로 칸이 비면 재매수한다.
- 사이클에 집착하지 않는다: 7개 다 못 채워도, 매도/손절로 flat 되면 그 사이클은 끝난 것.


## 10. 실거래 전 검증 필요 (Open Questions / TODO)

1. **토스 CLS(LOC) 주문 실접수 확인** — 문서상 `LIMIT+CLS` = US LOC 지원. 실제 매수/매도 LOC 접수·체결 동작 소액 검증.
2. **저가 LOC 종가매도의 가격 밴드** — 지정가를 prevClose 대비 크게 낮췄을 때 토스가 거부하는지(가격 밴드/틱). 거부되면 `stopSellDiscount`를 조정하거나 정정(modify) 전략 필요.
3. **동일 종목 매수 LOC + 매도 LOC 동시 접수** — 계좌 제약 없는지.
4. **결제 지연(T+1)** 반영된 매수가능금액 API 응답 타이밍.
5. **부분체결·미체결 조회** — order-history/order-info로 체결수량 정확히 확인.

## 11. 안전장치 (실주문 3중 가드, v4 관례 계승)

- `config.enabled = false` 기본.
- 환경변수 `DDSAOPAL_LIVE_ORDERS=YES_REALLY` 없으면 DRY-RUN(주문 미제출, 계획만 로그).
- DRY-RUN이 코드 기본값.
- 멱등성: 주문마다 `clientOrderId`(예: `<cycleSeq>-<date>-<lotId|buy>`)로 재시도 중복 방지.
